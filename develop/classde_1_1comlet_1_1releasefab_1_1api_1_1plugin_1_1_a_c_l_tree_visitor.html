Detailed Description

Visitor Pattern! Abstract class which is used for tree traversal.

Code for traversing over the tree is always the same, but the action is changing. Possible actions are the removal or addition of components. This abstract class allows the parameterization of this process. Generics are used because the parameters can change depending on the action. E.g. the addition of a delivery has an object of CCLDelivery as P2 and returns a boolean. The action to find a component by its ID has an Integer as P2 and returns a CCLComponent.
